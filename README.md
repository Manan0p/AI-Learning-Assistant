# AI Learning Assistant Backend

Production-ready FastAPI backend for the Get Set Skilled AI Engineering Intern Assignment. It generates personalized learning roadmaps, recommends portfolio projects, and answers learner questions with retrieval-augmented generation over stored roadmap chunks.

## Architecture Overview

The app is fully asynchronous and organized by responsibility:

- `app/api`: FastAPI route handlers and dependency wiring.
- `app/core`: settings, logging, and exception handling.
- `app/db`: SQLAlchemy async engine/session setup.
- `app/models`: SQLAlchemy 2.0 ORM models.
- `app/schemas`: Pydantic v2 request/response and LLM-output schemas.
- `app/repositories`: database access layer.
- `app/services`: business logic for Gemini, embeddings, RAG, progress, caching, and health.
- `app/prompts`: prompt templates.
- `app/utils`: JSON extraction and markdown rendering helpers.
- `tests`: unit and integration tests with mocked Gemini behavior.

## AI Tools & Frameworks Used

- **Google Gemini 2.5 Flash**: Core LLM for generating JSON roadmaps, projects, and chat responses.
- **Google `gemini-embedding-2`**: Text embedding model used for RAG.
- **LangChain**: Used purely as a utility for text chunking (`RecursiveCharacterTextSplitter`).
- **pgvector**: Native PostgreSQL extension for efficient storage and cosine-similarity retrieval of vector embeddings.

## API Endpoints

Base path: `/api/v1`

- `POST /roadmap`: generates, validates, persists, chunks, embeds, and stores a personalized roadmap.
- `POST /project`: generates and persists one realistic portfolio project from either a `roadmap_id` or `goal_title + skills`.
- `POST /chat`: embeds the learner query, retrieves top 5 roadmap chunks by cosine similarity, and answers using only retrieved context.
- `GET /progress?roadmap_id=...`: returns roadmap progress items and completion percentage.
- `PATCH /progress`: updates one progress item to `pending`, `in_progress`, or `completed`.
- `GET /health`: checks database and Redis connectivity.
- `GET /health` at the root: lightweight process health check.

FastAPI automatically exposes OpenAPI documentation at `/docs` and `/openapi.json`.

## Database Schema

`roadmaps`

- Stores learner input, validated roadmap JSON, markdown rendering, and timestamps.

`roadmap_chunks`

- Stores chunk text and a `vector(768)` embedding generated by `gemini-embedding-2`.
- Indexed with HNSW using `vector_cosine_ops`.

`projects`

- Stores generated portfolio project JSON with optional roadmap linkage.

`chat_messages`

- Stores user/assistant turns, follow-up questions, and retrieved chunk IDs.

`progress_items`

- Stores per-phase completion state for each roadmap.

## RAG Design

The chat endpoint does not inject the entire roadmap into the prompt. It follows a real retrieval workflow:

1. Embed the user query with Google `gemini-embedding-2` using retrieval-query mode.
2. Search `roadmap_chunks` for the top 5 chunks ordered by pgvector cosine distance.
3. Build a compact prompt containing only those retrieved chunks.
4. Generate a JSON response with Gemini 2.5 Flash.
5. Persist the chat turn and the retrieved chunk IDs for traceability.

## Chunking Strategy

Roadmaps are converted to markdown first because headings, bullets, milestones, and weekly plans create semantically useful boundaries. The chunker targets about 400 tokens by using roughly 1,600 characters per chunk with a 300-character overlap, which approximates a 75-token overlap. This keeps each chunk focused while preserving continuity across phase boundaries.

## Prompt Engineering Decisions

- Prompts require JSON-only output with no markdown fences.
- Each generation task includes the exact Pydantic JSON schema.
- The roadmap prompt emphasizes realistic cadence, measurable milestones, and portfolio orientation.
- The project prompt asks for one specific achievable project, not a list.
- The chat prompt instructs Gemini to answer only from retrieved context and admit missing context when necessary.
- Malformed LLM output is parsed defensively and retried up to 3 times with exponential backoff.

## Setup

1. Copy the environment file:

```bash
cp .env.example .env
```

2. Set `GOOGLE_API_KEY` in `.env`.

3. Start the stack:

```bash
docker compose up --build
```

4. Open the API docs:

```text
http://localhost:8000/docs
```

The API container runs Alembic migrations on startup.

## Local Development

With Poetry:

```bash
poetry install
poetry run alembic upgrade head
poetry run uvicorn app.main:app --reload
```

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Assumptions

- `gemini-embedding-2` embeddings are stored as 768-dimensional vectors.
- A progress item maps to each generated roadmap phase.
- Redis caching is opportunistic: cache failures are logged at debug level and do not break core generation.
- Integration tests avoid live PostgreSQL, Redis, and Google calls; production behavior is covered through service boundaries and mocks.
